# 카카오톡 챗봇 베타 입점 미팅 준비 자료

> 미팅 일시: 목요일 1시 30분 (구글밋, Q&A 방식)

---

## 1. 봇 개요 한 줄 요약

**"주식왕"** — 실제 한국 주식 시세를 기반으로 가상 자금으로 투자를 연습하는 **게이미피케이션 가상 투자 시뮬레이터** 카카오톡 챗봇

---

## 2. 예상 질문 & 답변

---

### Q1. 이 챗봇의 목적과 대상 사용자는 누구인가요?

**답변:**
- **목적**: 실제 돈 없이 한국 주식 투자를 안전하게 연습할 수 있는 교육·엔터테인먼트 서비스
- **대상 사용자**:
  - 주식 투자에 관심은 있지만 실제 투자를 망설이는 초보자
  - 새로운 투자 전략을 실전 시세로 테스트하고 싶은 중급 투자자
  - 친구·동료와 투자 실력을 경쟁하고 싶은 MZ세대
- **핵심 가치**: "게임처럼 재미있게, 실전처럼 사실적으로" — 게이미피케이션 요소를 통해 투자 학습 진입 장벽을 낮춤

---

### Q2. 기존 카카오페이증권 주식봇과 뭐가 다른가요?

| 구분 | 카카오페이증권 주식봇 | 주식왕 |
|------|----------------------|--------|
| **핵심 컨셉** | 정보 조회 도구 (AI 요약) | 게이미피케이션 투자 시뮬레이터 |
| **사용 환경** | 팀채팅방 @멘션 방식 | 전용 챗봇 채널 (스킬 서버) |
| **모의투자** | 월 1회, 1,000만원 고정 | 상시 가능, 실시간 매수·매도, 포트폴리오 영구 관리 |
| **거래 방식** | 단순 모의투자 | 실시간 시세 기반 매수/매도/전량매수/전량매도, 수수료 0.1% 적용 |
| **포트폴리오** | 없음 (개인 투자 정보 미저장) | 개인별 보유 종목, 평균 단가, 수익률, 자산 히스토리 차트 |
| **경쟁 요소** | 팀채팅방 내 공유 수준 | 전체 수익률 랭킹, PvP 배틀 (주가 예측 대결), 주간 챌린지 |
| **게임 요소** | 없음 | 출석 보상, 복권, 슬롯머신, 룰렛, 동전던지기, 하이로우 |
| **성장 시스템** | 없음 | 일간 미션, 주간 챌린지, 업적, 마일스톤, 연속 출석 보너스 |
| **뉴스** | AI 기업/매출 요약 | Google News RSS 기반 실시간 종목 뉴스 |
| **시장 정보** | 주식 지수, 종목 시세 | KOSPI/KOSDAQ 지수 + 급등주/급락주/거래량 TOP 10 |
| **AI 기능** | AI 요약 (기업, 매출, 종목토론방) | 없음 (순수 시세 데이터 기반) |
| **종목 추천** | 불가 | 불가 (정보 제공에 집중) |

**차별화 핵심 3가지:**
1. **"게임" ≠ "조회 도구"**: 주식봇은 정보 조회 도구이지만, 주식왕은 지속적 참여를 유도하는 게임형 서비스
2. **개인화된 영구 포트폴리오**: 유저별 가상 자금, 보유 종목, 거래 내역을 DB에 영구 저장하여 장기적 투자 시뮬레이션 가능
3. **소셜 경쟁**: 수익률 랭킹, PvP 배틀, 주간 챌린지 등으로 유저 간 경쟁과 리텐션 극대화

### Q3. 어떤 기술 스택을 사용하나요?

**답변:**
- **백엔드**: Python 3.11 + FastAPI (비동기 웹 프레임워크)
- **WSGI 서버**: Uvicorn (ASGI)
- **데이터베이스**: PostgreSQL (프로덕션), SQLite (로컬 개발)
- **ORM**: SQLAlchemy 2.x (커넥션 풀링, pool_pre_ping 활성)
- **캐싱**: cachetools (TTLCache) — 시세 캐시 60초, 랭킹 캐시 5분
- **외부 API**: 한국투자증권 KIS OpenAPI (실시간 주가), Google News RSS (뉴스)
- **컨테이너**: Docker (멀티스테이지 빌드, non-root 실행)
- **배포**: Railway (PaaS)

### Q4. 데이터베이스 구조는 어떻게 되나요?

**답변:**
총 9개 테이블:

| 테이블 | 역할 | 주요 컬럼 |
|--------|------|-----------|
| `users` | 유저 정보 | kakao_id(PK), nickname, cash, 출석/미션/업적 정보 |
| `holdings` | 보유 주식 | kakao_id(FK), stock_code, quantity, avg_price |
| `transactions` | 거래 내역 | trade_type(BUY/SELL), price, quantity, profit |
| `battles` | PvP 배틀 | challenger/opponent, stock, prediction, status |
| `weekly_challenges` | 주간 챌린지 정의 | challenge_type, target_value, reward |
| `user_challenges` | 유저별 챌린지 진행 | current_value, completed |
| `milestones` | 마일스톤 달성 | milestone_type, achieved_at |
| `asset_history` | 자산 히스토리 | record_date, total_asset, cash, stock_value |
| `stock_cache` | 종목 코드/이름 캐시 | stock_code(PK), stock_name |

**데이터 보존 정책:**
- 거래 내역: 90일 보존 후 자동 삭제
- 자산 히스토리: 365일 보존 후 자동 삭제
- 유저 데이터: 영구 보존 (삭제 요청 시 대응 가능)

### Q5. 서버 구조와 배포는 어떻게 하나요?

**답변:**
- **아키텍처**: 카카오 챗봇 관리자센터 → 스킬 서버 (POST /skill) → 명령어 핸들러 → 서비스 레이어 → DB
- **배포 플랫폼**: Railway (PaaS) — Docker 이미지 기반 자동 배포
- **서버 구성**: 단일 인스턴스, Uvicorn 1 Worker
- **스케일링**: 현재 단일 인스턴스로 충분, 유저 증가 시 Worker 수 증가 또는 수평 스케일링 가능
- **헬스체크**: `/health` 엔드포인트 (DB 연결 상태 포함), Docker HEALTHCHECK 30초 간격

---

### Q6. 유지보수는 어떻게 하나요?

**답변:**
- **모니터링**:
  - 헬스체크 엔드포인트 (`/health`) — UptimeRobot 등 외부 모니터링 서비스 연동 가능
  - 구조화된 로깅 시스템 (거래 감사 로그, 게임 감사 로그, 배틀 로그 분리)
  - 요청 ID 추적 (`X-Request-ID`) — 에러 디버깅용
- **장애 대응**:
  - KIS API 서킷 브레이커 (연속 5회 실패 시 60초 차단 후 자동 복구 시도)
  - DB 커넥션 풀 pre_ping (끊긴 연결 자동 재연결)
  - 커넥션 풀: 기본 5개 + 추가 10개, 30분마다 갱신
- **데이터 관리**:
  - 자동 DB 마이그레이션 (서버 시작 시 누락 컬럼 자동 추가)
  - 오래된 레코드 자동 정리 (거래내역 90일, 자산히스토리 365일)
  - 만료된 배틀 자동 정리 (서버 재시작 시)
- **업데이트**: Git push → Railway 자동 빌드·배포 (Docker 기반)
- **공휴일**: 2024~2026년 한국 공휴일 데이터 내장, 장 시간 자동 관리

### Q7. 보안은 어떻게 처리하나요?

**답변:**
- **Rate Limiting**: 유저당 분당 30회 요청 제한 (인메모리, 스레드 안전)
- **요청 크기 제한**: 최대 10KB (DoS 방지)
- **CORS**: 카카오톡 도메인만 허용 (`talk.kakao.com`, `pf.kakao.com`, `kapi.kakao.com`)
- **입력 검증**:
  - kakao_id: 100자 제한, 영숫자+하이픈+언더스코어만 허용
  - utterance: 500자 제한, Null 바이트·제어문자 제거
  - SQL 인젝션 방지: SQLAlchemy ORM 사용 (파라미터 바인딩)
  - DB 마이그레이션 SQL: 화이트리스트 기반 컬럼 타입 검증
- **관리자 API**: Bearer 토큰 인증, `secrets.compare_digest` (타이밍 공격 방지)
- **Docker 보안**: non-root 사용자(`appuser`)로 실행
- **금액 오버플로우 방지**: `safe_add`, `safe_subtract`, `safe_multiply` 헬퍼 (최대 10조원 상한)

### Q8. 개인정보는 어떻게 처리하나요?

**답변:**
- **수집하는 정보**: 카카오 유저 고유 ID (`kakao_id`), 닉네임 (카카오 OpenBuilder 제공 시)
- **수집하지 않는 정보**: 실명, 전화번호, 이메일, 실제 투자 정보
- **저장 위치**: Railway PostgreSQL (클라우드 DB)
- **로그 마스킹**: kakao_id 앞 4자리만 표시 (`abcd****`)
- **데이터 삭제**: 관리자 API로 전체 초기화 가능, 개별 유저 삭제 기능 추가 가능
- **목적 제한**: 오직 게임 진행 (가상 자산 관리)에만 사용

### Q11. 미니게임에 도박성 요소가 있지 않나요?

**답변:**
- **가상 화폐만 사용**: 모든 거래와 게임은 가상 자금(게임 내 화폐)으로만 진행됩니다. 실제 금전 거래나 환전은 없습니다.
- **장 마감 시간에만 가능**: 미니게임(슬롯, 룰렛 등)은 주식 장이 닫힌 시간에만 플레이 가능 (복권 제외)
- **일일 제한**: 복권은 1일 5회로 제한
- **기대값 관리**: 모든 게임의 기대값은 50%~150% 범위 내로 설계, 시스템이 자동 검증
  - 복권: ~90%, 슬롯: ~90%, 룰렛: ~90%, 동전: 100%, 하이로우: ~88%
- **목적**: 장 마감 후에도 유저 참여를 유지하기 위한 부가 콘텐츠이며, 투자 교육이 핵심 목적

### Q12. 카카오 챗봇 스킬 서버 연동 방식은?

**답변:**
- **연동 방식**: 카카오 i 오픈빌더 스킬 서버 (POST `/skill`)
- **응답 포맷**: Kakao Skill Response v2.0 준수
  - `simpleText` (텍스트)
  - `basicCard` (카드)
  - `listCard` (리스트)
  - `carousel` (캐러셀)
  - `quickReplies` (빠른 응답 버튼)
- **응답 시간**: KIS API 캐시(60초) 활용으로 대부분 1초 내 응답
- **명령어 체계**: `/명령어 [인자]` + 한글 초성 축약어 지원 (예: `/ㅅㅅ` = `/시세`)

### Q14. 확장성은 어떻게 대응하나요?

**답변:**
- **현재**: 단일 인스턴스로 소규모 사용자 대응 충분
- **단기**: Uvicorn Worker 수 증가 (2~4 Workers)
- **중기**: PostgreSQL 커넥션 풀 확장 (pool_size 조정), 시세 캐시 TTL 최적화
- **장기**:
  - Redis 캐시 도입 (다중 인스턴스 간 캐시 공유)
  - 수평 스케일링 (Railway 또는 AWS ECS)
  - 읽기 전용 DB 레플리카 (랭킹 등 읽기 부하 분산)

## 3. 핵심 기능 데모 시나리오 (미팅 시 보여줄 흐름)

```
1. /시작        → 가입 + 500만원 지급
2. /출석        → 매일 30만원 보상 (연속 출석 보너스)
3. /시세 삼성전자 → 실시간 시세 조회
4. /매수 삼성전자 10 → 10주 매수
5. /포트폴리오   → 보유 현황 + 수익률
6. /급등        → 급등주 TOP 10
7. /뉴스 삼성전자 → 관련 뉴스
8. /랭킹        → 수익률 TOP 10
9. /복권        → 미니게임 체험
10. /배틀생성 삼성전자 상승 → PvP 배틀
```

## 4. 기술 아키텍처 요약도

```
[카카오톡 사용자]
       │
       ▼
[카카오 i 오픈빌더]
       │ POST /skill
       ▼
[FastAPI 서버 (Railway/Docker)]
  ├── 미들웨어: Rate Limit, CORS, 요청 크기 제한, Request ID
  ├── CommandHandler (명령어 라우팅)
  │   ├── TradingHandlerMixin (매수/매도/포트폴리오)
  │   ├── GameHandlerMixin (복권/슬롯/동전/룰렛/하이로우)
  │   ├── MarketHandlerMixin (시세/급등/급락/뉴스/검색)
  │   └── SocialHandlerMixin (랭킹/배틀/챌린지/마일스톤)
```

## 5. 미팅 시 주의사항

1. **도박 우려 선제 대응**: "모든 게임은 가상 화폐, 실제 금전 거래 없음" 강조
2. **개인정보 최소 수집 강조**: kakao_id와 닉네임만 사용, 실제 투자 정보 미수집
3. **카카오페이증권 주식봇과 경쟁이 아닌 보완**: "정보 조회 vs 교육형 시뮬레이션"으로 포지셔닝
4. **종목 추천 안 함**: "투자 판단 보조 정보 제공에 집중, 직접적 추천 불가" 명시
5. **KIS API 라이선스**: 정식 등록된 오픈 API 사용임을 명확히
