# 도움말 카드 흐름 설계 (Help Card Flow)

> `/도움말` 응답을 **카테고리별 카드 캐러셀**로 설계한 문서입니다.
> 목적: ① 긴 본문 잘림 해결 ② 카드 → 기능 → 후속 기능으로 이어지는 **사용 흐름 유지**.

---

## 1. 배경 — 왜 바꿨나

기존 `/도움말`은 모든 명령어를 하나의 긴 `simpleText`로 출력했습니다.
카카오톡 `simpleText`는 길이 제한이 있어(약 1,000자) **본문이 중간에서 잘렸고**,
사용자에게는 "주식 투자 항목만 보이다가 갑자기 끊기는" 어색한 화면이 나왔습니다.

**해결:** 기능 카테고리별로 **`basicCard` 캐러셀**(좌우로 넘기는 카드)로 분리.
- 카드 1장 = 카테고리 1개 → 본문이 잘리지 않음
- 각 카드에 **바로 실행 버튼**을 배치 → 카드에서 기능으로 끊김 없이 진입

관련 코드:
- 카드 데이터: `config.py` → `Messages.HELP_CARDS`
- 렌더링: `handlers/command_handler.py` → `CommandHandler.handle_help()`
- 응답 빌더: `utils/kakao_response.py` → `KakaoResponse.carousel(..., card_type="basicCard")`
- 테스트: `tests/test_help_cards.py`

라우팅은 `_find_handler`가 `startswith`로 매칭하므로 `/도움말`, `/도움말주식` 등 모두 `handle_help`로 들어옵니다.

## 2. 카드 구성 (6장)

| # | 카드 | 본문(명령어) 요약 | 버튼 (→ 이동) |
|---|------|------------------|----------------|
| 1 | 🐜 만렙개미 시작하기 | 게임 소개 + "옆으로 넘기세요" 안내 | 🚀 `/시작` · 📅 `/출석` · 📈 `/급등` |
| 2 | 📊 주식 투자 | 급등·급락·인기·거래량·시세·뉴스·검색·시장·매수·매도 | 📈 `/급등` · 🔍 `/시세 삼성전자` · 💼 `/포트폴리오` |
| 3 | 💼 내 자산 | 잔고·포트폴리오·차트·거래내역·랭킹·내순위·각성랭킹·닉네임 | 💼 `/포트폴리오` · 🏆 `/랭킹` · 📊 `/급등` |
| 4 | 🎁 보상 · 🧬 각성 | 출석·보물상자·각성·능력 (장 마감 후 안내) | 📅 `/출석` · 🎁 `/보물상자` · 🧬 `/각성` |
| 5 | 🎮 예측 게임 | 시장예측·업다운 (장 마감 후 안내) | 🎮 `/시장예측` · 🔢 `/업다운` · 🎁 `/보물상자` |
| 6 | ⚔️ 소셜 | 배틀·배틀목록·미션·업적·챌린지·마일스톤 | ⚔️ `/배틀` · 🎯 `/미션` · 🏅 `/챌린지` |

> 버튼은 카드당 **최대 3개**(카카오 제한). 모든 버튼의 `messageText`는 실제 라우팅되는
> 명령어를 가리키며, 이는 테스트(`test_every_help_button_routes_to_handler`)로 강제합니다.

## 3. 사용 흐름 (Flow Graph)

도움말 캐러셀은 **허브**이고, 각 카드 버튼은 해당 도메인으로 진입합니다.
진입한 화면은 다시 자체 버튼/퀵리플라이를 가지고 있어 **흐름이 계속 이어집니다.**

```
                         ┌─────────────────────────┐
        /도움말  ───────▶│   도움말 카드 캐러셀(6장)   │
   (/도움말주식 등 포함)   └─────────────────────────┘
                                     │ (카드 버튼)
        ┌───────────────┬───────────┼───────────┬───────────────┐
        ▼               ▼           ▼           ▼               ▼
     /시작           /급등        /포트폴리오    /각성          /배틀
        │               │           │           │               │
        ▼               ▼           ▼           ▼               ▼
   가입/환영      급등주 TOP10    내 자산 현황   각성 정보      배틀 생성/목록
   버튼:시작/      버튼:급락/      버튼:랭킹/     버튼:출석/      버튼:배틀목록/
   도움말         ETF급등/시세    급등/출석      보물상자/각성   미션/챌린지
        │               │           │           │               │
        └───────────────┴─────► 각 화면의 버튼으로 인접 기능으로 계속 이동 ◄┘
```

### 흐름 설계 원칙
1. **허브-스포크:** `/도움말`(허브)에서 모든 핵심 도메인으로 1탭 진입.
2. **데드엔드 금지:** 모든 버튼은 유효한 명령어로만 연결(테스트로 보장). 끊기는 화면 없음.
3. **상호 연결:** 진입 화면은 인접 기능 버튼을 노출(예: 급등 ↔ 급락 ↔ ETF급등 ↔ 시세).
4. **컨텍스트 회귀:** 게임/각성처럼 장 마감 후만 가능한 기능은 카드·화면에 ⚠️ 안내로 회귀 경로 제공.

## 4. 카카오 플랫폼 제약 (설계 시 준수 사항)

| 항목 | 제한 | 본 설계 |
|------|------|---------|
| `simpleText` 본문 | 약 1,000자 (초과 시 잘림) | 사용 안 함 → 잘림 원인 제거 |
| `carousel` 카드 수 | 최대 10장 | 6장 |
| `basicCard` 버튼 | 카드당 최대 3개 | 3개 이하 |
| 캐러셀 지원 카드 타입 | `basicCard` / `commerceCard` 등 (`textCard` 미지원) | `basicCard` 사용 |
| 카드 본문 길이 | 단말에 따라 길면 잘릴 수 있음 | 카드당 **≤ 350자**(테스트로 강제), 실제 ≤ 145자 |

> ⚠️ 카드 본문/제목의 정확한 표시 한계는 단말·클라이언트 버전에 따라 다를 수 있어,
> 실제 카카오톡에서 한 번 육안 확인을 권장합니다. 길이 가드는 `test_help_card_descriptions_not_truncated`.

## 5. 수정·확장 방법

카드 내용/버튼은 전부 **`config.Messages.HELP_CARDS`** 한 곳에서 관리합니다(데이터 주도).

```python
HELP_CARDS = [
    {
        "title": "카드 제목",
        "description": "줄바꿈(\n)으로 명령어 나열 (약 200자 이내 권장)",
        "buttons": [
            {"label": "버튼명", "messageText": "/명령어"},  # 최대 3개
        ],
    },
    # ...
]
```

- 카드 추가/순서 변경: 리스트 항목을 추가/이동 (최대 10장).
- 버튼 추가: `buttons`에 항목 추가 (카드당 3개 이하 — 초과 시 테스트 실패).
- 새 버튼은 **반드시 라우팅되는 명령어**를 가리킬 것 (`COMMAND_ROUTES`에 존재해야 테스트 통과).
- `handle_help()` 코드는 수정 불필요 — 설정만 바꾸면 렌더링에 자동 반영됩니다.
